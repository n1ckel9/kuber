# Деплой сервера «Кубер» (постоянный адрес)

Цель: вынести API на хостинг, чтобы у него был неизменный `https://…`-адрес.
Тогда не нужен туннель, комп можно выключать, и APK пересобирается только ради
нативных изменений (а не из-за смены адреса).

## Вариант 1 — Render (бесплатно, проще всего)

### 1. Залить код на GitHub
```
git init            # если ещё не репозиторий
git add .
git commit -m "Кубер: initial"
```
Создай пустой репозиторий на github.com и запушь:
```
git remote add origin https://github.com/<логин>/vodovoz.git
git branch -M main
git push -u origin main
```

### 2. Развернуть на Render
1. Зарегистрируйся на https://render.com (можно через GitHub).
2. **New → Blueprint** → выбери свой репозиторий → Render прочитает `render.yaml`.
3. Дождись сборки. Получишь адрес вида `https://kuber-api.onrender.com`.
4. Проверь: открой `https://…onrender.com/api/health` → `{"ok":true}`.

### 3. Прописать адрес в приложение
- В `.env` и `eas.json` замени `EXPO_PUBLIC_API_URL` на адрес Render.
- Пересобери APK один раз (`eas build -p android --profile preview`) — дальше адрес не меняется.

## Важно про базу данных
Сейчас данные в SQLite-файле (`server/vodovoz.db`). На бесплатном Render диск
**эфемерный** — при передеплое/перезапуске данные обнуляются. Для теста это ок.
Для боевого запуска — подключить **PostgreSQL** (у Render есть бесплатный) или
платный диск. Запросы изолированы в `server/db.js`, миграция на Postgres локальна.

## Альтернативы Render
- **Railway** (railway.app) — похоже, есть бесплатный лимит.
- **Fly.io** — чуть сложнее, но щедрый бесплатный тариф + постоянный диск.
- **VPS** (условный timeweb/vscale) — полный контроль, ~символическая цена.
