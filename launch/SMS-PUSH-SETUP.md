# Подключение реального входа по SMS и пушей

Код полностью готов (env-gated). Ниже — что сделать на аккаунтах, чтобы включить «по-настоящему».

---

## 1. Реальный вход по коду

### Вариант А — SMS.ru (рекомендую на старте: работает для любого номера)
1. Регистрация на **sms.ru**, пополнить баланс (SMS ~2–4 ₽/шт).
2. (Опц.) одобрить **имя отправителя** — иначе шлётся с общего номера, это нормально для старта.
3. Скопировать **api_id** (Настройки → Программистам/API).
4. На прод-сервере в `server/.env`:
   ```
   SMSRU_API_ID=твой_api_id
   ```
5. Перезапустить: `sudo systemctl restart kuber`.
6. **Проверка:** запроси код входа в приложении — придёт реальный SMS. (В ответе сервера `devCode` пропадёт — значит боевой режим включился.)

### Вариант Б — Telegram Gateway (дешевле, но только у кого есть Telegram)
1. **gatewayapi.telegram.org** → получить токен.
2. В `.env`: `TELEGRAM_GATEWAY_TOKEN=...`

Можно включить **оба**: сначала пробуется Telegram, если у номера его нет — падает на SMS.ru. Приоритет: Telegram → WhatsApp → SMS → демо.

> Пока не задан ни один ключ — работает демо-режим (код показывается на экране). Для реальных людей это НЕ годится — обязательно подключи SMS.ru или Telegram.

## 2. Пуши на Android (FCM) — один раз

Пуши в собранном приложении на Android идут через Firebase Cloud Messaging. Настраивается один раз:

1. **console.firebase.google.com** → создать проект (напр. «Kuber»).
2. **Add app → Android** → package name: **`ru.vodovoz.services`** → зарегистрировать.
3. Скачать **`google-services.json`** → положить в **корень проекта** (рядом с `app.json`).
4. В `app.json`, в раздел `"android"`, добавить строку:
   ```json
   "googleServicesFile": "./google-services.json"
   ```
5. Firebase → **Project settings → Service accounts → Generate new private key** → скачать JSON.
6. В терминале проекта:
   ```
   npx eas-cli credentials
   ```
   → платформа **Android** → профиль (preview/production) → **Push Notifications: Manage** → **Google Service Account Key for FCM V1** → загрузить JSON из п.5.
7. Пересобрать: `npx eas-cli build -p android --profile preview`.
8. Установить новую сборку. При первом входе приложение попросит разрешение на уведомления и зарегистрирует токен.
9. **Проверка:** с другого аккаунта вызови событие (отклик на заказ / «выехал») → на первом устройстве придёт пуш. Тап по нему откроет нужный заказ.

**iOS-пуши** настраиваются EAS при сборке под Apple Developer-аккаунт (позже, вместе с App Store).

## Что уже сделано в коде (трогать не нужно)
- **Клиент:** запрашивает разрешение, получает Expo push-токен (по projectId), показывает уведомления в открытом приложении, тап открывает заказ.
- **Сервер:** `notify()` шлёт пуш через Expo Push API с `orderId` в payload.
- **SMS:** мультиканальная доставка кода (Telegram/WhatsApp/MAX/SMS.ru), автоприоритет и фолбэк, env-gated.

## Быстрый тест пуша без приложения
На сервере: `node server/testpush.js "ExponentPushToken[...]"` (токен берётся из логов при входе на устройстве).
